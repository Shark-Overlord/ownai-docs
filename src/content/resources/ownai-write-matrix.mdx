## 项目定位

OwnAI Write Matrix 是一个本地运行的小红书图文自动化系统。它不是单点写作工具，而是把“创作者画像、热点雷达、提示词策略、自动化任务、草稿箱发布”组织成一条完整内容生产链路。

它适合需要持续产出小红书图文内容的个人创作者、内容运营和 AI 自动化工作流搭建者。系统会把热点候选转化为标题、正文、标签、封面提示词和正文配图提示词，再进入草稿箱统一检查、生成图片和发布。

## 解决什么问题

- 把小红书账号定位、目标人群、痛点标签和内容支柱沉淀成可复用的创作者画像。
- 用热点雷达持续发现内容机会，减少每天手动找选题的重复劳动。
- 用提示词策略管理文章结构、内容类型和配图风格，避免每次从零开始写提示词。
- 自动生成小红书草稿，包括标题、正文、标签、封面提示词和正文图片提示词。
- 在草稿箱集中预览、编辑、删除、批量生成图片，并在人工检查后执行发布。

## 核心工作流

1. 创建创作者画像，维护账号定位、目标人群、痛点标签、内容支柱和写作规则。
2. 在热点雷达中配置关键词或按账号画像发现热点，形成候选选题。
3. 通过提示词调试选择内容类型和视觉风格，例如教程型、清单型、案例型、避坑型、复盘型和观点型。
4. 创建自动化任务，让系统按监控配置生成标题、正文、标签和配图提示词。
5. 草稿进入草稿箱后，先预览和修改，再生成图片、批量选择并发布到小红书。

## 界面与能力

### 创作者画像

创作者画像是整套系统的基础。它定义账号是谁、写给谁、解决什么问题、用什么风格表达，以及最终希望用户做什么。

页面把定位总览、目标人群、痛点标签、内容支柱、写作规则、样本证据、变现路径、禁区规则和提示词预览放在同一套档案里。后续热点筛选、提示词生成和自动化任务都会读取这份画像，保证内容不是孤立生成，而是围绕账号定位持续产出。

### 热点雷达

热点雷达负责发现创作者关注的实时热点。用户可以补充关键词、配置图像选题策略，并把匹配后的热点转入选题库，再进入文章生成和审核发布流程。

页面中的监控配置、雷达总览、热点池和选题转化对应从“发现热点”到“生成选题”的前半段链路。它的价值不是堆热点列表，而是用账号画像和关键词先筛掉不适合的方向。

### 提示词调试

提示词调试页面把内容类型、热点方式、图文生成模式、封面风格、配色方案、语气风格和转化目标拆开管理。调试模式不会入库，也不会推送审核队列，适合测试不同标签组合下的封面、图文页和正文提示词。

页面下方还有示例图库，能快速切换教程型、清单型、案例型、避坑型、复盘型、观点型等结构，也能选择手账便签风、蓝白科技、橙黄活力、红黑警示等视觉模板。

### 草稿箱

草稿箱是内容生产后的质检区。生成出的草稿会在这里按状态展示，并支持搜索标题、正文、标签和错误信息。

每条草稿可以查看图片数量、平台状态和小红书 ID，也可以执行预览、编辑、删除、生成图片成品和一键发布。对于批量生产场景，草稿箱能把“生成”和“发布”拆开，保留人工检查环节。

## 技术栈

- 后端：Python 3.11、FastAPI、SQLite
- 前端：Next.js、React、TypeScript、Tailwind CSS
- 自动化：Playwright、小红书 MCP 桥接
- AI 接口：OpenAI-compatible Chat API，支持 DeepSeek / NewAPI 等兼容转发
- 生图接口：OpenAI Images-compatible API，可在系统设置中配置

## 适合场景

- 做小红书账号矩阵，需要稳定沉淀选题、草稿和图片生成规则。
- 想把热点发现、标题正文生成、标签生成和图文卡片生成串成自动化流程。
- 希望保留人工审核，但减少每天重复复制提示词、整理草稿和生成图片的工作。
- 需要一个本地运行、配置可控、数据保存在本机的内容生产工作台。

## 使用提醒

- 项目用于学习、研究和个人自动化工作流，请遵守小红书平台规则。
- 发布能力依赖本地登录状态和页面结构，平台页面更新后可能需要同步适配。
- 自动发布前建议先在草稿箱人工检查标题、正文、标签和图片，确认内容质量后再执行发布。
